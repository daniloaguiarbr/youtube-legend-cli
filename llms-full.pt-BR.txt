# youtube-legend-cli (completo)

[English](llms-full.txt) | [Português Brasileiro](llms-full.pt-BR.txt)

> CLI Rust não interativa para baixar legendas do YouTube através de provedores terceirizados via uma interface Unix nativa de stdin/stdout. Binário estático único, sem daemon, sem telemetria.

## Arquitetura
- Parser de argumentos derivado de clap (Cli) com enums FormatArg e LanguageArg e sete flags globais (--config, --log-level, --log-format, --color, --no-progress, --dry-run, --yes)
- Despacho de top-level em commands::run seleciona batch vs URL única baseado em Cli::batch
- ProviderChain percorre ProviderA depois ProviderB em ordem, throttled para uma requisição por segundo
- Cada Provider implementa fetch_subtitle async retornando SubtitleInfo e fetch_content retornando Vec<u8>
- retry::retry_with_backoff envolve fetch_subtitle com três tentativas em 1s, 2s, 4s; honra Retry-After (delta-seconds ou data RFC 2822) com fallback de 60s limitado a 300s
- provider::robots verifica o robots.txt upstream antes de cada requisição; o caminho Disallow produz EX_UNAVAILABLE
- cache::cache_path resolve para ~/.cache/youtube-legend-cli/<author>/subtitles/<video_id>/<lang>/<format>.bin
- TTL do cache padrão de 24 horas e sobrescrevível via --cache-ttl
- crypto::encrypt_token produz uma string base64(hex(salt(32) || iv(16) || ciphertext)) com PBKDF2-HMAC-SHA1 em 100 iterações
- mimalloc é o alocador global em src/main.rs

## Superfície de API pública
- Cli — parser clap; Cli::validate() retorna Result<(), String> para checagens de sanidade de flags
- AppError — enum thiserror com variantes InvalidInput, InvalidUsage, InvalidUrl, NoSubtitle(NoSubtitleReason), ProviderUnavailable, RateLimited, StdinEmpty, Http, Io, Serde, Crypto, SubtitleTooLarge, Internal; #[non_exhaustive]
- AppResult<T> = Result<T, AppError>
- NoSubtitleReason — enum com NotPublished, NotAvailable, AutoGenerated, PaidOrMembers
- run(cli: Cli) -> AppResult<ExitCode> — entry point de top-level
- extract::run e batch::run — despacho de comandos
- convert_format, output_success, output_error — helpers de envelope JSON
- cache_path, default_ttl, read_cache, write_cache, invalidate_cache — API de cache
- retry_with_backoff, CircuitBreaker — helpers de retry
- extract_video_id, srt_to_text — helpers de parse
- is_stdin_tty, is_stdout_tty, read_url_from_stdin, read_urls_from_stdin, write_subtitle_to_stdout, write_to_stderr — helpers de IO
- normalize_nfc — Unicode NFC (text é pub(crate), alcançável apenas via o crate)
- encrypt_token — crypto
- init_tracing(cli_log_level, cli_log_format, cli_color, quiet) — setup de logging; honra as env vars YT_LOG_LEVEL e YT_LOG_FORMAT acima dos args de CLI
- Trait Provider com name(), fetch_subtitle, fetch_content
- Format::Srt, Format::Txt com as_str() e extension()
- SubtitleInfo com video_id, language, format, source_url, byte_size
- ProviderChain::new, with_min_interval, throttle, fetch_subtitle
- ProviderA, ProviderB — implementações concretas de provedor
- ProviderHeadless (feature = "headless") — fallback opcional de Chromium/Chrome

## Flags
- --lang: en, pt, es, fr, de, it, mais formas BCP 47 (pt-BR, pt_BR.UTF-8, EN-us). Padrão en. Malformado/não suportado → exit 2.
- --format: txt ou srt. Padrão txt.
- --timeout: timeout HTTP em segundos. Padrão 30.
- --verbose: emite eventos de tracing em stderr
- --quiet: suprime todo stderr que não seja erro
- --json: emite envelope JSON no stdout
- --batch: lê múltiplas URLs do stdin
- --user-agent: sobrescreve o User-Agent padrão (padrão é o nome do crate)
- --cache-ttl: TTL do cache em horas (padrão 24)
- --no-cache: pula leituras do cache
- --config <PATH>: caminho para um arquivo TOML sobrescrevendo os defaults abaixo; flags de CLI ganham em conflito
- --log-level <LEVEL>: error | warn | info | debug | trace. Padrão warn. Honrado via env var YT_LOG_LEVEL acima do valor de CLI.
- --log-format <FORMAT>: text | json. Padrão text. Honrado via env var YT_LOG_FORMAT acima do valor de CLI.
- --color <WHEN>: auto | always | never. Padrão auto. Auto usa isatty em stdout e respeita NO_COLOR.
- --no-progress: suprime quaisquer barras de progresso em stderr
- --dry-run: pula I/O de rede; serve leituras só do cache
- --yes: assume sim para qualquer confirmação

## Exit codes (BSD sysexits.h)
- 0 sucesso
- 64 uso ou entrada inválida (EX_USAGE)
- 65 URL inválida (EX_DATAERR)
- 66 nenhuma legenda para o vídeo (EX_NOINPUT)
- 69 todos os provedores indisponíveis, ou rate limited, ou robots.txt disallow (EX_UNAVAILABLE)
- 70 erro interno / IO / HTTP / timeout / crypto / serde (EX_SOFTWARE)
- 78 erro de configuração (EX_CONFIG)
- 130 recebido SIGINT ou SIGTERM

Em HTTP 429 a CLI honra o header Retry-After tanto em delta-seconds quanto no formato RFC 2822 HTTP-date (fallback 60s quando ausente, limitado a 300s) antes de tentar novamente.

## Cancelamento
- SIGINT e SIGTERM são cabeados via tokio_util::CancellationToken em main.rs
- o primeiro sinal dispara cancelamento cooperativo
- o segundo sinal força saída imediata do processo (std::process::exit(130))
- requisições em voo são abortadas no próximo await point
- processo sai com código 130
- a API async exposta por este crate é cancellation-safe em todo await point público

## Observabilidade
- tracing-subscriber com EnvFilter é inicializado de --log-level (CLI) > YT_LOG_LEVEL (env) > RUST_LOG (env) > default "error"
- 14 entry points de API pública carregam #[tracing::instrument] com nomes de campo explícitos
- cada tentativa de retry emite event = "retry" com número da tentativa e próximo delay
- saída JSON é habilitada por --log-format json ou por YT_LOG_FORMAT=json
- RUSTDOCFLAGS="-D warnings" é aplicado no CI para todo build de doc

## Segurança
- AES-256-CBC mais PBKDF2 para assinatura de token
- Limite em memória de 50 MiB no tamanho da legenda decodificada
- sem telemetria, sem analytics, sem chamada de rede para casa
- cookie store com reqwest rustls
- validação de entrada: parsing de URL, regex de 11 chars para video-id, sanitização de path de cache
- conformidade com robots.txt para ambos os provedores
- SIGILO: secret_endpoints é pub(crate); hostnames upstream nunca são visíveis no rustdoc ou baseline public-api
- SECURITY.md: tabela de versões suportadas, modelo de ameaça, canal privado de reporte
